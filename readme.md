# Mazes For Programmers

## Overview

Coding exercises and example code derived from the book "Mazes for Programmers" by Jamis Buck. This repository contains a small grid/maze library, several maze-generation algorithms, and demo scripts that render mazes to PNG files.

## Layout

- `basic_grid/` - core grid and cell implementation used by the algorithms.
- `maze_lib/` - maze generation algorithms (Aldous-Broder, Wilsons, Sidewinder, Binary Tree, etc.).
- `demo/` - runnable demo scripts you can execute to generate PNGs.
- `mazes/` - output images generated by the demos.

## Prerequisites

- Ruby (2.5+ recommended)
- A C compiler & development headers if any native gems are required for your system's Ruby (usually already present on macOS via Xcode command line tools).

## Installing gems (local project environment)

This project includes demos that depend on gems declared in `demo/Gemfile`. It's best to use Bundler so you can install dependencies locally for the project.

1. Install Bundler (if you don't have it already):

```bash
gem install bundler
```

2. From the repository root, install the gems used by the demos. To install into a project-local directory (so you don't pollute your system gems), run:

```bash
cd demo
bundle install --path vendor/bundle
```

This installs gems into `demo/vendor/bundle` and lets you run the demo scripts with `bundle exec`.

If you prefer to install gems into the system RubyGems, simply run `bundle install` without `--path`.

## Running the demos

From the repository root you can run any of the demo scripts in `demo/`. Examples:

- Generate a Wilsons maze and save to `mazes/wilsons.png`:

```bash
# if using bundle-managed gems
cd demo
bundle exec ruby wilsons_demo.rb
# or without bundler (if gems are installed globally)
ruby demo/wilsons_demo.rb
```

- Use the general `makemaze.rb` driver to build mazes with options:

```bash
ruby demo/makemaze.rb -t sidewinder -w 20 -h 20 -f mazes/sidewinder.png
```

After running a demo, generated images are saved in the `mazes/` directory (it will be created automatically by the demos if missing).

## Troubleshooting

- If you see errors about missing `chunky_png`, make sure you installed the demo gems with Bundler (see above). Then run demos with `bundle exec ruby ...` so the correct gemset is used.
- On macOS, if `gem install` fails due to native extensions, ensure Xcode command line tools are installed:

```bash
xcode-select --install
```

## Notes

- I recently fixed an issue in `maze_lib/wilsons.rb` where the implementation had syntax mistakes and did not properly mark cells as visited. The fixed implementation performs a loop-erased random walk from an unvisited cell until it reaches the visited set, then links the path into the maze. Running `demo/wilsons_demo.rb` should now terminate and save `mazes/wilsons.png`.
